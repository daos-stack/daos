enum daos_cont_props

DAOS container property types valid in rage (DAOS_PROP_CO_MIN, DAOS_PROP_CO_MAX).

Enumerator
DAOS_PROP_CO_LABEL 

Label - a string that a user can associated with a container. default = ""

DAOS_PROP_CO_LAYOUT_TYPE 

Layout type: unknown, POSIX, HDF5, Python, Database, Parquet, ... default value = DAOS_PROP_CO_LAYOUT_UNKNOWN

DAOS_PROP_CO_LAYOUT_VER 

Layout version: specific to middleware for interop. default = 1

DAOS_PROP_CO_CSUM 

Checksum on/off + checksum type (CRC16, CRC32, SHA-1 & SHA-2). default = DAOS_PROP_CO_CSUM_OFF

DAOS_PROP_CO_CSUM_CHUNK_SIZE 

Checksum chunk size default = 32K

DAOS_PROP_CO_CSUM_SERVER_VERIFY 

Checksum verification on server. Value = ON/OFF default = DAOS_PROP_CO_CSUM_SV_OFF

DAOS_PROP_CO_REDUN_FAC 

Redundancy factor: RF(n): Container I/O restricted after n faults. default = RF0 (DAOS_PROP_CO_REDUN_RF0)

DAOS_PROP_CO_REDUN_LVL 

Redundancy level: default fault domain level for placement. default = 1 (rank level)

DAOS_PROP_CO_SNAPSHOT_MAX 

Maximum number of snapshots to retain.

DAOS_PROP_CO_ACL 

ACL: access control list for container An ordered list of access control entries detailing user and group access privileges. Expected to be in the order: Owner, User(s), Group(s), Everyone

DAOS_PROP_CO_COMPRESS 

Determine whether inline compression is enabled Value: DAOS_PROP_CO_COMPRESS_OFF/LZ4/DEFLATE[1-4] Default: DAOS_PROP_CO_COMPRESS_OFF

DAOS_PROP_CO_ENCRYPT 

Determine whether encryption is enabled Value: DAOS_PROP_CO_ENCRYPT_OFF, DAOS_PROP_CO_ENCRYPT_AES_XTS{128,256}, DAOS_PROP_CO_ENCRYPT_AES_CBC{128,192,256}, DAOS_PROP_CO_ENCRYPT_AES_GCM{128,256} Default: DAOS_PROP_CO_ENCRYPT_OFF

DAOS_PROP_CO_OWNER 

The user who acts as the owner of the container. Format: user@[domain]

DAOS_PROP_CO_OWNER_GROUP 

The group that acts as the owner of the container. Format: group@[domain]

DAOS_PROP_CO_DEDUP 

Determine whether deduplication is enabled Require checksum to be enabled Value DAOS_PROP_CO_DEDUP_OFF/MEMCMP/HASH Default: DAOS_PROP_CO_DEDUP_OFF

DAOS_PROP_CO_DEDUP_THRESHOLD 

Deduplication threshold size Default: 4K

DAOS_PROP_CO_ROOTS 

First citizen objects of container, see daos_cont_root_oids

DAOS_PROP_CO_STATUS 

Container status Value "struct daos_co_status".

DAOS_PROP_CO_ALLOCED_OID 

OID value to start allocation from

DAOS_PROP_CO_EC_CELL_SZ 

EC cell size, it can overwrite DAOS_PROP_CO_EC_CELL_SZ of pool

◆ daos_prop_from_str()

int daos_prop_from_str ( const char *  str,
daos_size_t  len,
daos_prop_t **  prop 
)

Allocate a new property from a string buffer of property entries and values. That buffer has to be of the format: prop_entry_name1:value1;prop_entry_name2:value2;prop_entry_name3:value3; prop must be freed with daos_prop_free() to release allocated space. This supports properties that can be modified on container creation only: label, cksum, cksum_size, srv_cksum, dedup, dedup_threshold, compression, encryption, rf, ec_cell

Parameters
[in]strSerialized string of property entries and their values
[in]lenSerialized string length
[out]propProperty that is created

◆ daos_prop_set_ptr()

int daos_prop_set_ptr ( daos_prop_t *  prop,
uint32_t  type,
const void *  ptr,
daos_size_t  size 
)

Set the pointer value of a property entry in a property. The property type must expect that it's entry is of a pointer type. This duplicates the buffer internally and the entry buffer is freed with daos_free_prop(). The user does not need to keep the string buffer around after this function is called. If the entry already has a value set, it frees that and overwrites it with this new value.

Parameters
[in]propProperty list
[in]typeType of property to look for
[in]ptrPointer to value of entry to set
[in]sizeSize of value
