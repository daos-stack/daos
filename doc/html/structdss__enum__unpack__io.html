Detailed Description

Used by dss_enum_unpack to accumulate recxs that can be stored with a single VOS update.

ui_oid and ui_dkey are only filled by dss_enum_unpack for certain enumeration types, as commented after each field. Callers may fill ui_oid, for instance, when the enumeration type is VOS_ITER_DKEY, to pass the object ID to the callback.

ui_iods, ui_recxs_caps, and ui_sgls are arrays of the same capacity (ui_iods_cap) and length (ui_iods_len). That is, the iod in ui_iods[i] can hold at most ui_recxs_caps[i] recxs, which have their inline data described by ui_sgls[i]. ui_sgls is optional. If ui_iods[i].iod_recxs[j] has no inline data, then ui_sgls[i].sg_iovs[j] will be empty.
