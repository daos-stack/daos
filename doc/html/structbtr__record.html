union { ... }

Fix-size key can be stored in if it is small enough (DAOS_HKEY_MAX), or hashed key for variable-length/large key. In the later case, the hashed key can be used for efficient comparison.

When BTR_FEAT_UINT_KEY is set, no key callbacks are used for comparisons.

When BTR_FEAT_DIRECT_KEY is used, we store the umem offset of the relevant leaf node for direct key comparison

§ rec_off

umem_off_t rec_off

It could either be memory ID for the child node, or body of this record. The record body could be any of varous things:

  • the value address of KV record.
  • a structure includes both the variable-length key and value.
  • a complex data structure under this record, e.g. a sub-tree.
